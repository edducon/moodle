# Умный помощник для навигации по курсам Moodle

## Описание проекта
**Moodle Smart Assistant** - это браузерное расширение и серверная инфраструктура, созданные для улучшения пользовательского опыта для платформы LMS Московского Политехнического университета.

* **Для студентов:** упрощение навигации по материалам, быстрый поиск нужных заданий и лекций с помощью интерактивного чат-бота, а также появление своевременных уведомлений о приближающихся дедлайнах работ.
* **Для преподавателей:** приведение структуры различных курсов к единому стандарту оформления.

## Реализованный функционал

### 1. Клиентская часть
Расширение автоматически активируется на страницах и выполняет работу ключевых функций в фоновом режиме:
* **Сбор данных (Парсинг):** Скрипт анализирует DOM-дерево страницы курса. Извлекается ID курса, его название, а также список всех видимых тем и модулей (задания, тесты, файлы, форумы). Собранная структура отправляется на сервер для синхронизации.
* **Интерфейс ассистента:** На страницу внедряется плавающая кнопка и окно чата для взаимодействия с пользователем.
Добавлена функция анимации, которая позволяет плавно скроллить страницу к конкретному элементу курса (например, к нужной лабораторной работе) и визуально подсвечивать его по команде от бэкенда.

### 2. Серверная часть (REST API)
Бэкенд спроектирован с использованием FastAPI и обеспечивает быструю обработку запросов. Настроена политика CORS для корректного взаимодействия с браузерным расширением.

**Эндпоинты:**
* `POST /api/course/sync`: Эндпоинт для сохранения курса. Принимает JSON со структурой курса от клиента. Если курс уже существует в базе, его контент обновляется (Upsert), если нет - создается новая запись.
* `POST /api/chat`: Эндпоинт для общения с ассистентом. На данный момент реализована проверка наличия курса в базе и возврат тестового ответа с первичной аналитикой (подсчет количества тем).

### 3. База данных
Для хранения данных используется таблица `courses` со следующими полями:
* `course_id` (String) - уникальный идентификатор курса в системе Moodle.
* `title` (String) - полное название курса.
* `content` (JSONB) - гибкая структура, хранящая иерархию тем и модулей.
* `last_updated` (DateTime) - время последней синхронизации данных.

## Инструкция по локальному запуску

### Программные требования
* Установленный **Python 3.10+**
* Установленный и запущенный **PostgreSQL**

### Шаг 1
1. Склонируйте репозиторий и перейдите в папку с проектом.
2. Создайте виртуальное окружение и активируйте его:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Для Linux/macOS
   venv\Scripts\activate     # Для Windows
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Создайте базу данных в PostgreSQL (например, `moodle_bot_db`).
5. Переименуйте файл `env.example` в `.env` и укажите актуальные данные для подключения к вашей БД:
   ```env
   DATABASE_URL=postgresql://ваш_пользователь:ваш_пароль@localhost:5432/moodle_bot_db
   CORS_ORIGINS="*"
   ```
6. Запустите сервер:
   ```bash
   python main.py
   ```
   Сервер будет доступен по адресу: `http://127.0.0.1:8000` (Документация API: `http://127.0.0.1:8000/docs`).

### Шаг 2
1. Откройте браузер на базе Chromium (Google Chrome, Яндекс Браузер, Edge).
2. Перейдите на страницу управления расширениями: `chrome://extensions/`.
3. Включите **Режим разработчика** (Developer mode) в правом верхнем углу.
4. Нажмите кнопку **Загрузить распакованное расширение** (Load unpacked).
5. Выберите папку, содержащую файлы `manifest.json`, `content.js` и `style.css`.
6. Перейдите на `https://lms.mospolytech.ru/`, чтобы проверить работу расширения.
